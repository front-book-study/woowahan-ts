## 5.1] 조건부 타입

---

### 1. extends와 제네릭을 활용한 조건부 타입

extends는 타입을 확장할때 / 타입을 조건부로 설정할 때 / 제네릭 타입에서는 한정자 역할로 사용됨

```tsx
T extends U ? X : Y
// T를 U에 할당할 수 있으면 X 타입, 아니면 Y 타입

interface Bank {
 financialCode: string;
 companyName: string;
 name: string;
 fullName: string;
}

interface Card {
 financialCode: string;
 companyName: string;
 name: string;
 appCardType?: string;
}

type PayMethod<T> = T extends 'card' ? Card : Bank;
type CardPayMethodType = PayMethod<'card'>
type BankPayMethodType = PayMethod<'bank'>
```

### 2. 조건부 타입을 사용하지 않았을 때의 문제점

`제공된 예시 분석`

- 의도한 결과 :
  - useGetRegisteredList 는 param으로 넘긴 type에 따라서 api 호출을 하고
  - 그 결과값을 리턴하며 그 결과값은 type 파라미터에 대응되는 타입이다
  - type card / appcard = > Card
  - type bank ⇒ Bank
- 문제점 :
  - 리턴타입이 모두 PayMethodType[] 이다.
  - type 파라미터에 card, appcard, bank 어떤 값을 넣든 리턴 타입 같아서 구별하기 힘들다.
- 문제점의 원인 :
  - PayMethodType은 PayMethodInfo<Card> 와 PayMethodInfo<Bank> 의 유니온으로 설정되어있기 때문이다.
  - type 파라미터에 따라서 둘 중 어떤 타입의 결과값인지 타입스크립트는 알 수 없다.

### 3. extends 조건부 타입을 활용하여 개선하기

`위의 예시를 해결하는 방법`

- 제네릭 유니온으로 해결한다.
- 기존에 파라미터로 넘겨줬던 type을 제네릭으로 넘겨주고 이를 바탕으로 타입을 조건부로 Card, Bank를 리턴한다.
- 결과적으로 card, appCard ⇒ Card[] / bank ⇒ Bank[] 를 리턴한다.

```tsx
type PayMethodType<T extends 'card' | 'appcard' | 'bank'> = T extends
  | 'card'
  | 'appcard'
  ? Card
  : Bank;
```

### 4. infer를 활용해서 타입 추론하기

```tsx
type UnpackMenuNames<T extends ReadonlyArray<MenuItem>> =
  T extends ReadonlyArray<infer U>
    ? U extends MainMenu
      ? U['subMenus'] extends infer V
        ? V extends ReadonlyArray<SubMenu>
          ? UnpackMenuNames<V>
          : U['name']
        : never
      : U extends SubMenu
      ? U['name']
      : never
    : never;

export type PermissionNames = UnpackMenuNames<typeof menuList>;
// [기기 내역 관리, 헬멧 인증 관리, 운행 관리]
```

- menuList ⇒ as const이기 때문에 ReadonlyArray<MenuItem> 타입이다.
- T extends ReadonlyArray<infer U> ⇒ `U = MenuItem` 임을 알 수 있다.
- U extend MainMenu
  - MenuItem은 MainMenu | SubMenu 이기 때문에 분기처리한다.
  - true : U extends MainMenu
    - MainMenu[’subMenus’] 가 있으면 그 타입은 SubMenu[] ⇒ `V = SubMenu[]`
    - V extends ReadonlyArray<SubMenu> ⇒ `V = SubMenu[]`
    - true :
    - false
  - false : U extends SubMenu
    - true :
    - false : never → 해당 경우의 수는 없다.

## 5.4] 불변 객체 타입으로 활용하기

---

- 객체를 이용해서 타입을 정의
  - 객체 타입 : `typeof obj`
  - 객체의 키 타입 : `keyof typeof obj`
  - 객체의 value 타입 : `typeof obj[keyof typeof obj]`

### 일반 객체일 때,

> **key는 리터럴 타입, value는 가장 일반적인 원시타입으로 추론**한다.

```tsx
export const colors = {
  red: "#F45452",
  green: "#0C952A",
  blue: "#1A7CFF",
  black: "#000000",
  gray: "#222222",
  white: "#FFFFFF",
  mint: "#2AC1BC",
} ****

export type ColorsObjType = typeof colors;
// type ColorsObjType = {
//     red: string;
//     green: string;
//     blue: string;
//     black: string;
//     gray: string;
//     white: string;
//     mint: string;
// }

export type ColorsObjKeyType = keyof ColorsObjType;
// type ColorsObjKeyType = "red" | "green" | "blue" | "black" | "gray" | "white" | "mint"

export type ColorsObjValueType = ColorsObjType[ColorsObjKeyType];
// type ColorsObjValueType = string;
```

### 불변 객체일 때,

> 불변 객체는 readonly 객체.
> **`as const`** 키워드를 객체 뒤에 붙임으로써 해당 객체를 불변하게 만들 수 있음.
> **key, value 타입이 모두 리터럴 타입이 됨!**

```tsx
export const colors = {
  red: "#F45452",
  green: "#0C952A",
  blue: "#1A7CFF",
  black: "#000000",
  gray: "#222222",
  white: "#FFFFFF",
  mint: "#2AC1BC",
} **as const;**

export type ColorsObjType = typeof colors;
// type ColorsObjType = {
//     readonly red: "#F45452";
//     readonly green: "#0C952A";
//     readonly blue: "#1A7CFF";
//     readonly black: "#000000";
//     readonly gray: "#222222";
//     readonly white: "#FFFFFF";
//     readonly mint: "#2AC1BC";
// }

export type ColorsObjKeyType = keyof ColorsObjType;
// type ColorsObjKeyType = "red" | "green" | "blue" | "black" | "gray" | "white" | "mint"

export type ColorsObjValueType = ColorsObjType[ColorsObjKeyType];
// type ColorsObjValueType = "#F45452" | "#0C952A" | "#1A7CFF" | "#000000" | "#222222" | "#FFFFFF" | "#2AC1BC"
```

## 5.5] Record 원시 타입 키 개선하기

---

### 무한한 키를 집합으로 가지는 Record

> 불변객체가 아닌 경우 객체는 일반적으로 속성을 추가하고 값을 변경할 수 있다.
> 따라서 Record를 써서 객체의 타입을 정의하는 경우 string / number을 사용한다.
> 하지만, 이렇게 정의하는 경우 \*\*존재하지 않는 키값에 접근하더라도 에러를 내뱉지 않아서 의도치않은 런타임에러가 발생하기 쉽다.

⇒ 옵셔널 체이닝을 이용해서 null, undefined일 때도 안전하게 접근할 수 있도록 한다.\*\*

>

```tsx
type Category = string;

interface Food {
  name: string;
  // ...
}

export const foodByCategory: Record<Category, Food[]> = {
  한식: [{ name: '제육덮밥' }, { name: '뚝배기 불고기' }],
  일식: [{ name: '초밥' }, { name: '텐동' }],
};
```

```tsx
import { foodByCategory } from "../constants/food";

export const Food = () => {
  **const westernFood = foodByCategory["양식"];**

  return <div>{**westernFood.map**((food) => food.name)}</div>;
};
```

### 유한한 키를 가지는 Record : Unit 타입으로 변경하기

> 유한한 키가 존재하는 경우 유닛 타입을 이용해서 존재하지 않는 키로 접근하지 못하게 막는다.

```tsx
**type Category = '한식' | '일식' ;**
interface Food {
  name: string;
  // ...
}

**// 한식, 일식으로 키값 제한**
export const foodByCategory: **Record<Category, Food[]>** = {
  한식: [{ name: "제육덮밥" }, { name: "뚝배기 불고기" }],
  일식: [{ name: "초밥" }, { name: "텐동" }],
};
```

```tsx
import { foodByCategory } from "../constants/food";

export const Food = () => {
  const westernFood = **foodByCategory["양식"];**

  return <div>{westernFood?.map((food) => food.name)}</div>;
};
```

### 무한한 키를 가지는 Record : Partial 활용하여 정확한 타입 표현하기

---

> Partial를 Record과 같이 활용하게 되면 객체의 속성값이 undefined일 수 있어서 해당 상황을 고려할 수 있도록 해준다.
>
> - Record : 존재하지 않는 속성에 접근했을 때 undefined일 수 있는 상황을 고려하지 않아서 에러를 발생시키지 않는다.
> - PartialRecord : 존재하지 않는 속성에 접근했을 떄 undefined일 수 있으므로 해당 처리를 하라고 에러를 뱉는다.
>
> ⇒ PartialRecord 라는 커스텀 유틸타입을 만들어서 사용하는 것을 추천한다.

```tsx
**type PartialRecord<K extends string, T> = Partial<Record<K, T>>;**

export const foodByCategory: **PartialRecord<Category, Food[]>** = {
  한식: [{ name: "제육덮밥" }, { name: "뚝배기 불고기" }],
  일식: [{ name: "초밥" }, { name: "텐동" }],
};
```

```tsx
import { foodByCategory } from "../constants/food";

export const Food = () => {
  const westernFood = foodByCategory["양식"];

  return <div>{**westernFood**.map((food) => food.name)}</div>;
};

// Error
// 'westernFood' is possibly 'undefined'.ts(18048)
```

옵셔널 체이닝을 추가해서 해결할 수 있다.

```tsx
 return <div>{**westernFood?.**map((food) => food.name)}</div>;
```
